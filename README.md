## 🚀 Ce que j'ai construis

### 🎵 Carnet de Chants (dans la vraie vie)
> **L'app qui fait chanter les églises — littéralement.**

Application Android de gestion d'hymnes pour les églises protestantes évangéliques.  
Déployée en production et utilisée  avec un répertoire de **268 cantiques**.

🔧 **Stack technique :**
`Kotlin` · `Jetpack Compose` · `Room` · `Hilt` · `Ktor` · `Media3 / ExoPlayer` · `Clean Architecture` · `MkDocs`

✨ **Points forts :**
- Lecture audio intégrée (MP3 hébergés) avec migration complète vers Media3/ExoPlayer
- Architecture Clean avec injection de dépendances (Hilt)
- API REST avec Ktor pour la synchronisation des données
- Documentation technique exploitable générée avec MkDocs
- De vrais utilisateurs, de vrais retours, de vraies itérations

## 🌱 Mon parcours — de la salle de formation au code

Avant de me lancer dans le développement Android, j'ai été **formatrice pendant 30 ans**. 🎤  
30 ans à transmettre, à vulgariser, à concevoir des parcours pédagogiques — et surtout, à voir la technologie transformer la façon dont on apprend... et dont on enseigne.

Un licenciement = une opportunité, j'ai décidé de passer de l'autre côté de l'écran. 💡

**Pourquoi Android ?** Parce que le smartphone est devenu le premier outil d'apprentissage au monde. Concevoir des **modules de formation pensés pour le mobile** — pas adaptés après coup, mais conçus *nativement* pour le téléphone — c'est le défi qui m'anime.

Mon approche de dev, c'est celle d'une formatrice reconvertie :
- 🏗️ Partir de **vrais besoins** (ma communauté, mon église, des associations)
- 🧩 Comprendre l'architecture **avant** de coder
- 🔄 Itérer, casser, déboguer, recommencer — et **documenter** tout ça
- 🎯 Viser la **production**, pas juste le prototype
- 📚 30 ans de pédagogie = une obsession pour l'**expérience utilisateur**

J'ai commencé avec Java et MVVM, puis j'ai fait la transition vers **Kotlin + Compose** — et je ne regrette rien ! 🎉
